폰과 컨트롤러

폰이나 월드 내 플레이어에 대한 물리적 표현

폰이란 : 플레이어 혹은 컴퓨터가 조종할 수 있는 액터의 종류

object - actor - pawn
폰의 작동 방식 : 플레이어 혹은 ai의 controller가 pawn에 possess해 사용한다.

player controller는 영혼
pawn는 육체라고 생각하면 좋음

player controller는 플레이어의 입력을 받는 기능을 하는 actor이다.


각각을 ai controller에 빙의된 pawn으로 만들고  player pawn이 ai controller들에게 명령을 뿌린다.

나나이트 메쉬가 무엇인지
나나이트 : 
- 언리얼 5부터 도입된 새로운 메시 폴리곤 최적화 기술
- 나나이트가 적용된 메시는 폴리곤 개수와 최적화와 렌더링 성능 관련 최적화를 알아서 해줌
- 스태틱 메시에만 적용되어 스켈레탈 메시는 적용이 안됨
- 나나이트 메시를 사용할 경우 일반 스테틱 메시와 차지하는 디스크 크기는 같지만, 메모리 크기는 훨씬 적게 사용함
- 트라이앵글을 더 효율적으로 적게, 압축하여 사용하기 때문이다.


LOD:
나나이트의 원리 개념은 LOD 개념에서 시작한다.

멀리있는 메시와 가까이 있는 메시간의 표현을 다르게 한다.
가까이 있는 메시는 세밀하게 표현 멀리는 대충 표현
-> 이를 다이나믹하게 runtime에 함.



나나이트의 장점 :
포토 리얼스틱한 데이터를 때려 박아도 속도/메모리의 문제가 없다.
디자이너들이 로우 폴리 모델을 따로 만들 필요 없음
맵의 퀄 상승 (시네마급) 성능 좋아짐, 개발 용이
대폭 증가한 지오메트리의 복잡도, 리얼타임에서 이전보다 많은 트라이앵글 및 오브젝트 지원
프레임 예산이 폴리곤 수, 메모리 사용등에 의해 제한 받지 않음
-----하지만 모든 상황에 좋은 건 아님(안좋은 상황은 모름)-----


나나이트를 사용해야 하는 메시 타입
- 트라이앵글이 많거나 화면상에 표시되는 트라이앵글이 아주 작은 경우
- 씬 내에 인스턴스가 많은 경우
- 다른 나나이트 지오메트리의 주요 오클루더 역할을 하는 경우
- 버추얼 새도 맵을 사용하여 새도를 캐스팅하는 경우



나나이트 메시는 본질적으로 트라이엥글 매시이며,해당 데이터에 LOD와 압축이 적용되어 있다.
완전히 새로운 시스템을 활용하여 데이터 포맷을 놀랍도록 효율적인 방식으로 렌더링함.

멀티스레딩:
스텍의 사용이 다름 / 코드와 공유 데이터는 같음 

멀티스레딩의 장점:
장점
    시스템 자원 소모가 감소하여 자원의 효율성이 증대
    프로세스를 생성하여 자원을 할당하는 시스템 콜이 줄어 자원을 효율적으로 관리할 수 있다
    시스템 처리율 향상 (처리 비용 감소)
    스레드 간 데이터를 주고받는 것이 간단해지고 시스템 자원 소모가 줄어든다
    스레드 사이 작업량이 적어 Context Switching이 빠름 (캐시 메모리를 비울 필요가 없음)
    간단한 통신 방법으로 프로그램 응답시간 단축
    스레드는 프로세스 내 스택 영역을 제외한 메모리 영역을 공유하기에 통신 비용이 적다
    힙 영역을 공유하므로 데이터를 주고받을 수 있음
단점
    자원을 공유하기에 동기화 문제가 발생할 수 있음 (병목현상, 데드락 등)
    하나의 스레드에 문제가 발생하면 전체 프로세스에 영향을 끼친다
    주의깊은 설계가 필요하고 디버깅이 어렵다 (불필요 부분까지 동기화하면, 대기시간으로 인해 성능 저하 발생)
    단일 프로세스 시스템의 경우 효과를 기대하기 어렵다
